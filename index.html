Architecture
Each Scroll is one file.

No frameworks. No build step. No dependencies. A single HTML document that contains the tool, the data layer, the design system, and the demo engine.

~400KB per Scroll

Complete app suite including all sub-tools, styles, scripts, and demo sequences in a single file.

Tab Navigation Shell

Shared sidebar/tab UI switches between sub-tools. Cmd+K command palette for instant access.

IndexedDB Storage

All data lives in the browser. Zero server calls. Export/import JSON for full portability.

Standalone Extract

Any sub-tool can be exported as its own independent HTML file with one click.

Demo Engine

Self-playing demo mode built in. Screen-capturable. Hostable. No manual recording needed.

Unified Design System

Each Scroll has a category accent color. All sub-tools share typography, spacing, components.

Works Everywhere

Any browser. Any OS. Online or offline. From a USB stick, email attachment, or shared folder.

E2E Encrypted

When connected, all sync is end-to-end encrypted. Protocol layer planned for Scrolls that discover each other.

Parent App Blueprint
Single HTML. Complete suite.

Each parent Scroll follows this exact architecture pattern. One file. All tools. Zero dependencies.

ARCHITECTURE — EACH PARENT SCROLL
scroll-[name].html
├── Shell — nav bar, tab switcher, Cmd+K palette, settings, theme toggle
├── Sub-Tool Modules — each tool is a JS closure, lazy-loaded on tab switch
├── Design Tokens — CSS variables: category color, typography, spacing, radius
├── Data Layer — IndexedDB wrapper, JSON import/export, localStorage fallback
├── Demo Engine — auto-play sequences for each sub-tool, screen-capture ready
├── Extract Mode — export any sub-tool as standalone HTML
└── Share Layer — file-native sharing: download, AirDrop, email, embed link
Tab switching: show/hide DOM sections — no router
Sub-tool isolation: each tool is an IIFE with scoped state
Target size: <500KB including all sub-tools
Browser support: Chrome, Safari, Firefox, Edge — last 2 versions

Manifesto

The web was supposed to be documents. Somewhere along the way, every simple thing became a platform with a login screen, a pricing page, and a terms-of-service longer than the tool itself.

We're going back. A Scroll is a single file that contains everything — the tool, the data, the design. You own it the way you own a photograph or a letter. No server can shut it down. No company can revoke access. No subscription can expire.

Seven files. Eighty tools. Zero dependencies. This is software that belongs to you.
