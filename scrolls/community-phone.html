A Design Vision

The Open Intranet

Standalone, serverless HTML tools — designed as a coherent ecosystem.
The internet Steve Jobs would have wanted.

The Idea

Not a Store.
An Ecosystem.

We're not building templates to sell on Etsy. We're building a family of beautifully designed, standalone HTML tools that share a unified design language — each one self-contained, offline-capable, account-free, and connected by nothing but shared branding and a philosophy.

"Design is not just what it looks like and feels like. Design is how it works."

— Steve Jobs

The goal: own the design layer across an entire class of tools that currently live in ugly spreadsheets, fragmented SaaS, and walled-garden platforms. Monetization comes later. The brand comes first.

Design Principles

What Makes This Different

◯

Zero Dependencies

No accounts. No installs. No subscriptions. A single HTML file that opens and works — on any device, offline, forever.

◈

Owned Design Language

Every tool in the ecosystem shares typography, color, motion, and interaction patterns. Recognizable at a glance. Unmistakably ours.

◇

Privacy by Architecture

Data lives on the user's device. No cloud. No telemetry. No "we may share with partners." Local-first by design, not by policy.

▽

Standalone, Then Connected

Each tool is complete on its own. But the ecosystem is designed so that tools can eventually discover and talk to each other — via Doccoin or similar protocol.

□

Interactivity > Static

Full JavaScript. Timers, calculators, live scoring, animations, charts. Things a spreadsheet or PDF can never do.

△

Free at the Point of Use

No paywall on day one. Build the library, build the brand, build the network. Revenue follows reach.

Future Architecture

Standalone First.
Connected Later.

Every tool works on its own, forever. But the long game is a protocol layer — Doccoin or similar — that lets standalone HTML documents discover, reference, and transact with each other. An intranet that nobody owns and everybody uses.

1

Phase: Library

Build 30+ tools under unified branding. Each is a standalone .html file. No backend, no accounts, no dependencies.

2

Phase: Recognition

The design language becomes the signal. Users recognize the ecosystem — "this is one of those tools" — before any formal connection exists.

3

Phase: Protocol

Doccoin or equivalent: a lightweight protocol that lets HTML documents link, share state, and optionally transact. The intranet emerges from the documents themselves.

4

Phase: Monetization

The brand, the library, and the network are the asset. Revenue models — licensing, premium tiers, protocol fees — become available without being forced.

The Asset

We Own the Design Layer

Anyone can build an HTML tool. What we're building is the only coherent, branded ecosystem of them — with a shared design language, a growing library, and a future connection protocol.

Brand Gravity

Network Potential

When standalone tools can discover each other, the ecosystem becomes more valuable than the sum of its parts. That's the real moat.
