Naming & Category

What do we
call these things?

The technical name is Serverless HTML Solutions — accurate, credible, explains the architecture. But for the world, we need something people can say in a sentence, text to a friend, and remember. Something that feels like a format, not a product.

★

Scrolls

Recommended. Ancient wisdom meets modern web. "Send me the travel scroll." "There's a scroll for that." A scroll is a single document that contains everything — the tool, the data, the design. It evokes libraries, knowledge, portability. Technical enough to feel real, cultural enough to feel cool. "Powered by Pan Tech Scrolls"

◎

Zines

Self-published, grassroots, countercultural. "I made a zine for tracking our trip." Echoes punk/DIY culture. The risk: too niche, implies static content, not interactive tools.

◎

Panes

Window panes — you look through them to see and do things. "Open the finance pane." Clean, modern, technical without being cold. Risk: sounds too much like "pain."

◎

Drops

Like music drops, sneaker drops. "New drop just landed: a full project manager." Creates urgency and culture. Risk: overused in crypto/NFT space, might feel disposable.

Decision needed: Lock the name this week. Every solution, every demo video, every social post, every line of copy needs to use one consistent term. The name IS the brand. "There's a Scroll for that" could be our "There's an app for that."

Demo Automation

Hardcode the demo.
Automate the wow.

Every working solution needs a demo — but recording manually doesn't scale. The play: hardcode simulated user actions directly into the Foundry output, creating self-playing demos that can be hosted as iframes, embedded on landing pages, or screen-captured for social content.

1

Script the action

Define a 15–30 second sequence of user actions for each solution — clicks, typing, scrolling, state changes. This becomes the demo script.

2

Hardcode into HTML

Embed the action sequence as a JavaScript animation loop directly in the solution's HTML. Uses setTimeout, CSS transitions, and DOM manipulation to simulate real usage. No external deps.

3

Host as iframe

Deploy the self-playing demo HTML to Azure Static Web Apps. Embed as an iframe on the catalog site, landing pages, or partner sites. Visitors see a live, interactive demo without downloading anything.

4

Capture for social

Screen capture the self-playing demo for TikTok, Reels, Twitter/X, LinkedIn. 15-second clips of a solution working itself — no narration needed. Batch-produce 30+ demo videos in a day.

ImplementationHowAzure CostOutput
Demo Mode Flag Add ?demo=true URL param to any solution. When present, triggers the hardcoded action sequence instead of waiting for user input. $0 (static) Self-playing solution
Foundry Template Add a "Generate Demo Script" step to the Forge workflow. AI generates the JS animation sequence based on the solution's UI structure. ~$0.02/demo (API call) Automated demo generation
Iframe Embed Kit One-line embed code: <iframe src="scroll.html?demo=true">. Add to catalog site, partner pages, blog posts. $0 (static hosting) Embeddable demos everywhere
Batch Video Pipeline Headless browser (Playwright on Azure VM) opens each demo URL, records 15-sec screen capture, exports as MP4/GIF. Batch all 30+ solutions in one run. ~$5–10/batch 30+ social-ready clips
Revenue Architecture

Three paths to
money.

The original strategy said "let revenue emerge." That's still true for the ecosystem — but we now have specific mechanisms that can generate revenue within the 128-day window. These aren't distractions from the mission; they're aligned with it.

Revenue Stream 1

AI Credit Sales

The AI Integration Hub connects AI capabilities to any solution via custom API. Users consume AI credits for features like: generating content, analyzing data, auto-filling forms, smart suggestions. We sell the credits. This is a metered utility model — pay-per-use AI that powers the solutions they already love.

How it works: User buys a credit pack ($5/50 credits, $20/250, $50/750). Credits are consumed when any AI-powered feature fires inside any solution. We proxy through Azure OpenAI and mark up the cost. Margin: 40–60%.

Revenue Stream 2

Podcaster Toolkits

Free solutions get them in the door. Premium bundles get them paying. A "Podcaster Pro Kit" with 8–10 solutions pre-configured for podcast workflows: episode planner → show notes generator → social card maker → analytics dashboard. Bundle for $29–$49 one-time or $9/mo for AI features.

Revenue Stream 3

Custom Solution Building

The Forge proves we can build solutions fast. Offer custom solution building as a service: "Tell us what you need, we'll build you a Scroll." $200–$2,000 per custom solution depending on complexity. Target: small businesses, teams, niche communities.

Revenue target shift: The original plan said "$0 revenue target." Updated position: $0 revenue requirement, but active revenue experiments. If AI credit sales generate $500/month by June, that validates the model and funds post-Azure operations. We're not chasing revenue — we're testing revenue while building the library.

The Intake Engine

Solution ideation
at scale.

You mentioned "once we hone out the product feature intake engine concept." Here's what that looks like: a systematic workflow that turns any problem statement into a shippable solution in under 24 hours.

1

Problem capture

Someone identifies a problem: from social media complaints, podcaster feedback, internal needs, or community requests. Logged in the Intake Tracker (itself a solution).

2

Forge generation

Problem → Forge V2 → AI generates the solution with design system tokens, mobile responsive, offline-capable, demo mode baked in. 80% done in 10 minutes.

3

Human polish

20% human touch: UX refinement, edge cases, copy, specific domain knowledge. Quality Gate pass. This is where taste lives — AI can't do this part.

4

Ship + demo + distribute

Solution → catalog. Demo video auto-generated. Social post queued. Iframe embed code ready. If it's for a podcaster, send directly. If it has AI features, credits system integrated. Done.
